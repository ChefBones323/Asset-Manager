REPLIT: BUILD CIVIC MISSION CONTROL INTELLIGENCE LAYER

The platform uses a deterministic event-sourced architecture where all system state derives from the EventStore.

We will now implement the **Mission Control Intelligence Layer**, which adds three major capabilities:

1. SYSTEM TIME MACHINE (historical replay viewer)
2. PRINT AND EXPORT ENGINE (audit and reporting tools)
3. CIVIC INTELLIGENCE AI OPERATOR (system analysis assistant)

All functionality must integrate with the existing Mission Control dashboard.

The system must remain deterministic and auditable.

--------------------------------------------------

SYSTEM TIME MACHINE

Create a new page:

pages/SystemTimeline.tsx

Purpose:

Allow operators to explore the entire system timeline and reconstruct system state at any historical event.

--------------------------------------------------

TIMELINE VISUALIZATION

Display a horizontal timeline of system events.

Event types include:

content_created  
proposal_created  
proposal_approved  
policy_activated  
config_changed  
worker_heartbeat  
platform_post_sent  
platform_publish_failed  

Each event appears as a node on the timeline.

Color coding:

content = blue  
trust = green  
governance = purple  
worker = orange  
error = red  

--------------------------------------------------

TIMELINE CONTROLS

Add interactive controls:

timeline slider  
jump to event ID  
jump to timestamp  
step forward/back  

Moving the slider reconstructs system state from the EventStore.

This must be read-only.

It must never mutate live state.
