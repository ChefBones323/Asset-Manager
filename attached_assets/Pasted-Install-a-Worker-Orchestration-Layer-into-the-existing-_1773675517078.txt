Install a **Worker Orchestration Layer** into the existing OpenClaw Operator Runtime.

The system already has:
- AgentRuntime
- ToolRegistry
- ToolRouter
- PolicyGuard
- SchedulerService
- ExecutionEngine
- EventStore
- Civic Mission Control dashboard

The governance pipeline already exists:

proposal → approval → execution_engine → worker → eventstore

Your job is to **implement the worker layer and job queue** so that all approved proposals are executed asynchronously by worker processes.

CRITICAL RULES

1. Do NOT bypass the governance pipeline.
2. Workers must claim jobs from a queue instead of executing tasks directly.
3. All execution events must be emitted to the EventStore.
4. The dashboard must be able to query worker and queue status.

→ JobQueue

WORKER EXECUTOR

Create a loop that runs continuously.

Worker startup:

register_worker()

Worker loop:

heartbeat()

job = claim_job()

if job exists:

update worker status busy

execute tool via ToolRouter

update job status

emit event to EventStore

set worker idle

sleep(poll_interval)
