REPLIT: IMPLEMENT MINIMAL OPENCLAW OPERATOR INTEGRATION

This repository already contains a working deterministic civic platform with:

• Mission Control UI
• Execution Engine
• Governance approval pipeline
• Worker system
• EventStore
• CommandPalette
• replay/export services

DO NOT rebuild these systems.

Your task is to install a **minimal AI operator runtime layer** on top of the existing architecture.

The operator must:

• plan tasks
• select tools
• execute actions
• store memory
• respect approval policies
• schedule tasks

All execution must pass through the existing deterministic pipeline.

NEVER allow the agent to mutate system state directly.

All side-effects must flow through:

proposal → approval → execution engine → worker → eventstore

STEP 2 — CREATE CORE OPERATOR FILES
---------------------------------------------------

Create:

server/agent_runtime/agent_runtime.py

Purpose:
Main OpenClaw-style runtime loop.

Implementation outline:

class AgentRuntime:

    def __init__(self):
        load prompts
        load tools
        initialize memory

    def run_task(self, user_input):

        context = build_runtime_context(user_input)

        for step in range(max_iterations):

            plan = generate_plan(context)

            tool_request = select_tool(plan)

            result = execute_tool(tool_request)

            context = update_context(context, result)

            if task_complete(context):
                break
