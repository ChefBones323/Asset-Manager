# Contributing to Asset Manager

Thank you for helping improve Asset Manager.

## Workflow

- Work in a branch and keep changes small and incremental
- Preserve existing architecture and naming unless a clear improvement
- Avoid breaking routes, exports, replay functionality, or dashboard views
- Use environment variables and never commit secrets

## Code standards

- TypeScript: prefer strict, typed APIs and reusable components
- Backend: keep services modular and safe
- Frontend: maintain mission-control UI consistency

## Pull requests

Include:

- What changed and why
- Files touched
- Any new env vars / migrations
- Testing steps
