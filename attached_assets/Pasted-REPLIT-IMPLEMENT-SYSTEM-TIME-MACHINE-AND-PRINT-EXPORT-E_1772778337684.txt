REPLIT: IMPLEMENT SYSTEM TIME MACHINE AND PRINT / EXPORT ENGINE

The platform uses a deterministic event-sourced architecture where all state derives from the EventStore.

We will now add two major capabilities:

1. SYSTEM TIME MACHINE (historical replay viewer)
2. PRINT AND EXPORT ENGINE (for reports and audits)

These tools must integrate with the existing Mission Control interface.

All system state must remain deterministic and derived from the EventStore.

--------------------------------------------------

FEATURE 1: SYSTEM TIME MACHINE

Create a new page:

pages/SystemTimeline.tsx

Purpose:

Allow operators to explore and reconstruct the system state at any historical event.

The Time Machine must visualize the full system timeline.

--------------------------------------------------

TIMELINE UI

Display a horizontal timeline of system events.

Event types include:

content_created
proposal_created
proposal_approved
policy_activated
config_changed
worker_heartbeat

Each event appears as a node on the timeline.

Color coding must match system domains:

content = blue  
trust = green  
governance = purple  
worker = orange  
error = red  

--------------------------------------------------

TIMELINE CONTROLS

Add controls to allow operators to move through time.

Controls:

timeline slider  
jump to event ID  
jump to timestamp  
step forward/back  

When the slider moves, the system reconstructs state from the event log.

--------------------------------------------------

REPLAY PREVIEW

Create a replay preview panel showing reconstructed state.

Panels should include:

Feed state  
Trust graph state  
Governance policy state  
Worker configuration state  

The preview must not modify live state.

It is a read-only reconstruction using the event replay engine.

--------------------------------------------------

BACKEND REPLAY SERVICE

Create:

services/replay/replayService.ts

Function:

reconstructState(event_id)

This service queries the EventStore and rebuilds projections up to the chosen event.

--------------------------------------------------

EVENT FILTERING

Allow filtering timeline events by:

domain  
actor_id  
event_type

System Timeline
